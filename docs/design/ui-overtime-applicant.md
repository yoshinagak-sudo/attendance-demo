# 残業申請 申請者UI 設計メモ

最終更新: 2026-04-26

## 対象画面

1. `/overtime` 申請者エントリ（選択 + 履歴）
2. `/overtime/new` 新規申請フォーム（ステップ式）
3. `/overtime/[id]` 申請詳細

## デザイン方針

- 既存トークン（globals.css）に完全準拠。新色追加なし
- 余白: 4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 ルール厳守
- フォントサイズは globals.css の 11 / 12 / 13 / 14 / 16 / 20 / 22 / 28 のみ
- アクセント色は primary 1色 + warn / danger のみ
- 既存 `.container`, `.header`, `.section`, `.card`, `.badge`, `.table` を最大限再利用
- 残業申請固有の追加クラスは `globals.css` 末尾に `/* ===== 残業申請 ===== */` で集約

│ └────┘ └────┘ └────┘                              │

```

### 状態

- `?actor` 未指定: 履歴セクションは「申請者を選んでください」の empty state
- `?actor` 指定 + 履歴 0 件: 「まだ申請がありません」の dashed empty
- `?withdrawn=1`: 上部に薄 primary 帯トースト（4秒で消える）

### バッジ色（status）

| status | クラス | ラベル | 色 |
|---|---|---|---|
| submitted | `badge ot-badge-submitted` | 申請中 | primary-soft / primary-hover |
| approved | `badge ot-badge-approved` | 承認済 | primary-soft / primary-hover |
| rejected | `badge ot-badge-rejected` | 却下 | danger-bg / danger |
| sent_back | `badge ot-badge-sent-back` | 差戻 | warn-bg / warn |

submitted と approved の差: ドット色を変えて区別（submitted は muted 抜け、approved は塗り）

## 画面2: `/overtime/new` 新規申請フォーム

ステップ式（同一URL、`useState` で `step: 1 | 2`）。クライアントコンポーネントで管理。

### ステップインジケータ（最上部）

```

 ●            ○          step=1 で●○、step=2 で●●、現状はライン色も変わる

### バリデーション（クライアント側、ステップ1→2 移行時）

- 申請日: 必須、`YYYY-MM-DD` 形式
- 開始/終了時刻: 必須、終了 > 開始
- 現場名: 必須、50文字以内
- 作業内容: 必須、200文字以内
- 200文字超過: 文字カウンタ赤化 + ステップ進行不可
- 180文字超: 文字カウンタ warn 色

エラー時はステップ進めずフィールド下に小さく赤文字。

### サーバーサイド処理

- 送信時 `useActionState(createOvertimeRequest, null)` を呼ぶ
- 成功: Server Action 内で `redirect('/overtime/[id]?submitted=1')`
- 失敗: ActionResult のエラーをフィールド横に表示、ステップ1に戻る

### hidden field

- `userId` (URLの `actor` 由来)
- `startAt` / `endAt` ISO 形式（hidden として送信、表示は time input）
- `workSiteId` (datalist で既存サイト選択時)

実装: `combineDateAndTimeJST(workDate, hhmm).toISOString()` をクライアント側で組み立てて hidden input に書く。React の `value` バインディングで都度更新。

```

├────────────────────────────────────────────────────┤
│ ✓ 申請を送信しました   ← ?submitted=1 時のみ      │
│                                                    │
│ [申請中]   2026-04-26 提出                         │
│                                                    │
│ ┌──────────────────────────────────────────────┐ │
│ │ 申請者       田中 太郎                        │ │
│ │ 申請種別     事前申請                         │ │
│ │ 申請日       2026年4月26日（日）             │ │
│ │ 残業時間     18:00 〜 20:00 （2時間00分）   │ │
│ │ 現場名       第3工場ライン                   │ │
│ │ 作業内容     設備の定期点検…                 │ │
│ └──────────────────────────────────────────────┘ │
│                                                    │
│ ※ 差戻時のみ ↓                                    │
│ ┌──────────────────────────────────────────────┐ │
│ │ ⚠ 差戻コメント                               │ │
│ │ 工事内容を具体化してください                 │ │
│ └──────────────────────────────────────────────┘ │
│                                                    │
│ [申請を取り消す]    ← submitted の自分の申請のみ  │
│                                                    │
│ ※ sent_back の場合: 再申請フォーム展開            │
│ （前回値プリフィルした overtime-form と同形）     │
└────────────────────────────────────────────────────┘
```

### 親申請（再申請の場合）

- `parentId !== null` → 上部に「これは再申請です（前回: 4月20日の申請 →）」リンク

### アクション可否

| status | 自分の申請 | 表示するアクション |
|---|---|---|
| submitted | yes | 「申請を取り消す」（withdraw）|
| submitted | no | 何も出さない |
| approved | - | コメント表示のみ |
| rejected | - | 却下コメント表示のみ |
| sent_back | yes | 差戻コメント + 再申請フォーム |
| sent_back | no | 差戻コメントのみ |

「自分の」判定: query `?actor=<userId>` と `target.userId` 一致

## アクセシビリティ

- セグメント・ステッパは `role="radiogroup"` / `aria-checked`
- 警告帯は `role="status"` + `aria-live="polite"`
- フォームエラーは `aria-invalid="true"` + `aria-describedby="..."`
- 「次へ→」「申請を送信する」「← 修正する」など全て `<button type="submit">` か `<button type="button">` を明示
- 申請取り消し・再申請は `<form>` ベースなのでキーボードのみで完結
